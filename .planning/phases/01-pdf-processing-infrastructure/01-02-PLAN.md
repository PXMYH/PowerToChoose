  <action>
1. Create api/services/__init__.py (empty).

2. Create api/services/downloader.py with:
   - `_url_to_cache_path(url: str) -> Path`: Hash URL with hashlib.sha256, use first 16 hex chars + ".pdf" as filename under settings.CACHE_DIR. This ensures deterministic, collision-resistant filenames.
   - `async def download_pdf(url: str) -> DownloadResult`:
     - Check if cache file exists. If yes, return DownloadResult(cached=True, success=True, file_path=str(cache_path)).
     - If not cached, download using httpx.AsyncClient with:
       - timeout=settings.PDF_DOWNLOAD_TIMEOUT
       - follow_redirects=True
       - Wrap in tenacity @retry decorator: wait_exponential(multiplier=1, min=2, max=30), stop_after_attempt(settings.PDF_DOWNLOAD_MAX_RETRIES), retry on httpx.HTTPStatusError (5xx only) and httpx.TransportError.
     - Stream response to temp file, then rename to cache path (atomic write).
     - On success: return DownloadResult(cached=False, success=True, file_path=...).
     - On failure (after retries exhausted): return DownloadResult(success=False, error=str(e)).
   - Do NOT retry on 4xx errors (client errors like 404 are permanent failures).

3. Create api/tests/__init__.py (empty).

4. Create api/tests/test_downloader.py with pytest tests:
   - Use httpx MockTransport or respx library for mocking HTTP responses (prefer respx if available, otherwise use a simple mock approach with monkeypatch).
   - Use tmp_path fixture for cache directory (override settings.CACHE_DIR in tests).
   - Test: successful download creates file and returns success=True, cached=False.
   - Test: second download of same URL returns cached=True without HTTP call.
   - Test: 404 returns success=False immediately (no retry).
   - Test: 500 triggers retries then returns success=False.
   - Test: cache path is SHA256-based and deterministic.
