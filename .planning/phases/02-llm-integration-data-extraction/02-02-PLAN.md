  <behavior>
    - extract_efl_data(text) returns EFLData when LLM returns valid structured data
    - extract_efl_data(text) retries on rate limit errors (HTTP 429) with exponential backoff
    - extract_efl_data(text) raises ExtractionError on non-retryable failures
    - extract_efl_data(text) passes the EFL text in a system+user prompt to the LLM
    - The prompt instructs the LLM to extract ALL EFLData fields, separate TDU from provider charges, and return null for fields not found
    - Extraction timeout respects settings.LLM_TIMEOUT
  </behavior>
  <action>
    1. Create api/services/efl_extractor.py with:

       a. A custom ExtractionError exception class.

       b. A SYSTEM_PROMPT constant that instructs the LLM:
          - You are an expert at parsing Texas Electricity Facts Labels (EFLs)
          - Extract all pricing, charges, contract, and provider details
          - Prices at 500/1000/2000 kWh are typically in cents/kWh -- convert to dollars/kWh (divide by 100) if the EFL shows cents
          - TDU delivery charges must be separated from provider energy charges
          - If a field is not present in the document, return null
          - plan_type is "fixed" if the rate is locked for the contract term, "variable" otherwise
          - For early termination fee, include both the dollar amount and any conditions

       c. An `async def extract_efl_data(efl_text: str) -> EFLData` function that:
          - Gets the instructor client via get_llm_client()
          - Calls client with model=settings.LLM_MODEL, response_model=EFLData,
            messages=[system prompt, user prompt with efl_text], timeout=settings.LLM_TIMEOUT
          - instructor handles Pydantic validation automatically (per LLM-04)
          - Returns EFLData on success
          - Raises ExtractionError with context on failure

       d. Wrap the LLM call with tenacity retry (per LLM-03):
          ```python
          from tenacity import retry, stop_after_attempt, wait_exponential, retry_if_exception_type
          import litellm

          @retry(
              retry=retry_if_exception_type((litellm.RateLimitError,)),
              wait=wait_exponential(multiplier=1, min=2, max=60),
              stop=stop_after_attempt(settings.LLM_MAX_RETRIES),
              reraise=True,
          )
          ```
          Note: tenacity is already a dependency from Phase 1. The retry decorator goes on an inner function that makes the actual LLM call, so that the outer async function can handle the final error gracefully.

       e. Since instructor.from_litellm returns a sync client, use `asyncio.to_thread()` to run the LLM call without blocking the event loop. The pattern:
          ```python
          import asyncio

          async def extract_efl_data(efl_text: str) -> EFLData:
              try:
                  result = await asyncio.to_thread(_call_llm, efl_text)
                  return result
              except Exception as e:
                  raise ExtractionError(f"EFL extraction failed: {e}") from e

          @retry(...)
          def _call_llm(efl_text: str) -> EFLData:
              client = get_llm_client()
              return client(
                  model=settings.LLM_MODEL,
                  response_model=EFLData,
                  messages=[
                      {"role": "system", "content": SYSTEM_PROMPT},
                      {"role": "user", "content": f"Extract structured data from this EFL document:\n\n{efl_text}"},
                  ],
                  timeout=settings.LLM_TIMEOUT,
              )
          ```

    2. Create api/tests/test_efl_extractor.py:
       - Mock `services.llm_client.get_llm_client` to return a mock client
       - The mock client should return EFLData instances directly (instructor does the Pydantic construction)
       - Test successful extraction: mock returns valid EFLData, verify all fields present
       - Test rate limit retry: mock raises litellm.RateLimitError twice then succeeds, verify retries happened
       - Test non-retryable failure: mock raises generic Exception, verify ExtractionError raised
       - Test that the system prompt contains key instructions (TDU separation, cents-to-dollars, null for missing)

    IMPORTANT: Do NOT call the real OpenRouter API in tests. All LLM calls must be mocked.
