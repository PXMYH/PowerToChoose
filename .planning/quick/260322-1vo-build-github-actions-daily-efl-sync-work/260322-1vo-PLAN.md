# Quick Plan 260322-1vo: GitHub Actions Daily EFL Sync

## Task
Build a GitHub Actions workflow that runs daily to fetch all plans from PTC API, extract EFL URLs, POST them to /api/efl/process/batch, and poll until jobs complete.

## Tasks

### Task 1: Create daily-efl-sync.yml workflow

**files:** `.github/workflows/daily-efl-sync.yml`
**action:**
- Cron schedule: daily at 6am UTC
- Manual trigger support (workflow_dispatch) with optional zip_code input
- Steps:
  1. Fetch plans from PTC API (`/api/plans?zip_code=78665&estimated_use=1000`)
  2. Extract plan IDs and EFL URLs from response
  3. Filter to plans that have valid EFL URLs (non-empty, ends in .pdf or contains known patterns)
  4. POST to `/api/efl/process/batch` with filtered plans
  5. Poll `/api/efl/status/{job_id}` for each job until all complete or timeout (30min)
  6. Report summary: completed, failed, timed out counts
- Use `VITE_API_URL` repo variable for base URL (already configured)
- Pure bash + curl + jq — no custom actions needed

**verify:** Workflow YAML is valid, uses correct API endpoints, has proper error handling
**done:** Workflow file exists at .github/workflows/daily-efl-sync.yml
